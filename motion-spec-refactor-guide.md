# Motion-Spec Metamodel Refactor Guide

This guide lists the breaking metamodel changes other components must follow.
The metamodels do not keep compatibility aliases for the old terms.

## Quantity Roles

The private `task/value-role` / `task/quantity-role` vocabulary was removed.
Do not type quantities as `Measured`, `Declared`, `Computed`, `Reference`,
`Error`, or `Commanded`.

Use the domain relation that gives the quantity its role:

| Old role | New representation |
|---|---|
| `Measured` | A normal `qudt:Quantity` generated by a sensor/runtime source. Use PROV (`prov:wasGeneratedBy`) if provenance is needed. |
| `Declared` | A normal authored `qudt:Quantity`; no RDF role type. |
| `Computed` | The output edge of the operator/evaluator that computes it; optionally `prov:wasGeneratedBy`. |
| `Snapshot` / `Sampled` | `snap:Snapshot` with `snap:snapshot-of` and optional `snap:sampled-on`. |
| `Reference` | Consumer relation such as `cstr:reference-value`, `cstr-hdl-ext:reference-signal`, or a domain-specific input edge. |
| `Error` | `cstr-hdl:error` or `cstr-hdl:error-signal`. |
| `Commanded` | `cstr-hdl:control-signal`; for forwarding use `slv-ext:command-signal`. |

## Snapshot

Snapshot moved from the metamodel root to `task/snapshot`.

Use:

- namespace: `https://secorolab.github.io/metamodels/task/snapshot#`
- class: `snap:Snapshot`
- source predicate: `snap:snapshot-of`
- clock predicate: `snap:sampled-on`
- clock values: `snap:task-clock`, `snap:entry-clock`

`snap:snapshot-of` is the precise runtime copy/sample relation and is declared
as `rdfs:subPropertyOf prov:wasDerivedFrom`.

## Command Forwarding

Command forwarding is no longer modeled as a solver algorithm. It is a driver
that forwards an already computed command quantity to a joint/actuator.

| Old term | New term |
|---|---|
| `slv-ext:CommandForwardingSolver` | `slv-ext:CommandForwardingDriver` |
| `slv-ext:CommandForwardingSpecification` | `slv-ext:ForwardedCommand` |
| `slv-ext:CommandForwardingAlgorithm` | removed |
| `slv-ext:command-forwarding` | `slv-ext:forwards-command` |
| `slv-ext:control-signal` | `slv-ext:command-signal` |

Do not attach `slv:solver` to a command-forwarding driver. Use the driver type
and `slv-ext:forwards-command` links.

## Constraint And Motion Guards

Guard aggregates are constraint-composition concepts:

- `cstr-ext:ConstraintConjunction`
- `cstr-ext:ConstraintDisjunction`
- `cstr-ext:has-constraint`

Do not use motion-spec extension terms for guard aggregation.

`mot:when` and `mot:until` may point to a leaf `cstr:Constraint` or one of
these aggregate constraint classes.

## Constraint Handler Extensions

Handler-only additions live in `cstr-hdl-ext`, not upstream `cstr-hdl` or
application metadata namespaces:

- `cstr-hdl-ext:control-mode`
- `cstr-hdl-ext:JointTorque`
- `cstr-hdl-ext:order`
- `cstr-hdl-ext:FeedForwardController`
- `cstr-hdl-ext:FeedbackController`
- `cstr-hdl-ext:reference-signal`

Use `cstr-hdl:evaluators`, not `cstr-hdl:constraint-evaluators`.

Duration and physical parameters are unit-bearing quantities. Use
`cstr-hdl-ext:debounce-duration`, not `debounce-seconds`.

## Trajectory And Profiles

Velocity profiles belong to `task/trajectory`:

- `traj:VelocityProfile`
- `traj:max-velocity`
- `traj:max-acceleration`
- `traj:max-jerk`
- `traj:measured-velocity`
- `traj:shape`

These quantity-valued properties must point to QUDT quantity nodes, not raw
numeric literals.

`traj:Lerp` also has `traj:profile` with values `Linear`, `EaseIn`, `EaseOut`,
or `EaseInOut`.

## Map Extension

Use the extension view classes for whole-vector views not covered by upstream
per-axis map views:

- `map-ext:PoseOrientationView`
- `map-ext:PosePositionView`
- `map-ext:WrenchVectorView`

The pose-orientation subspace is `map-ext:orientation`.
`map-ext:rotation` is reserved for the output predicate of
`map-ext:ComputeRotationFromPose`.

## Spatial Operators

`PoseDiffEvaluator` is a secorolab operator:

- class: `geom-op-ext:PoseDiffEvaluator`
- output predicate: `geom-op-ext:out`

Its inputs and output are `geom-coord:PoseCoordinate`. Do not model pose error
as `geom-rel:AccelerationTwist`.

## Solver Tuning

The scalar DLS/Tikhonov parameter remains `slv-ext:regularization`.

Physical limits are unit-bearing quantities:

- `slv-ext:torque-limit`
- `slv-ext:max-linear-accel`
- `slv-ext:max-angular-accel`

## JSON-LD Contexts

Use the scoped contexts in the metamodel JSON files. Class-specific properties
are intentionally scoped under their owning class terms; avoid relying on
flattened global property terms for those cases.
